# セットアップガイド（Claude内部参照用）

> このファイルは `/web-studio:setup` の実行ガイドです。
> ヒアリングの質問内容・分岐ロジックは `hearing-guide.md` に定義されています。
> 成果物は以下の4ファイルに書き出します。
> - `{project}/docs/interview.md`（Q&A生ログ + AI思考メモ）
> - `{project}/docs/context.md`（事業背景・判断基準）
> - `{project}/docs/requirements.md`（仕様・構成）
> - `{project}/docs/DESIGN.md`（デザイン方向性ドラフト）

---

## 起動時の判定

まず以下のファイルの存在を確認し、モードを決定する。
指示がない限りはヒアリングをスキップしないこと。事前に情報を渡された際は、その分の情報を記入し、不足分や深掘りたいところをヒヤリングするように。

| 状態 | モード |
|---|---|
| `docs/interview.md` がない | フルセットアップ → hearing-guide.md を読み込み、Pre-Stage から開始 |
| `docs/interview.md` はある（途中再開） | interview.md を読み込み、未回答ステージから再開 |
| `docs/context.md` と `docs/requirements.md` がある | 更新モード → ユーザーに選択肢を提示 |

**途中再開の方法:**
`interview.md` を読み込み、どのステージまで回答が記録されているかを確認する。
記録済みのステージは再度聞かない。未記録のステージから再開する。

**更新モードの選択肢:**
```
どちらを更新しますか？
1. 全体をやり直す
2. 事業・目的だけ更新する（context.md）
3. 仕様・要件だけ更新する（requirements.md）
4. デザイン方向性だけ更新する（DESIGN.md）
```

---

## ヒアリング実行

`${CLAUDE_SKILL_DIR}/references/hearing-guide.md` を読み込み（このファイルと同じディレクトリにある）、指示に従ってヒアリングを進める。

### ⚠ 重要：各ステージ完了後に即座にファイルへ書き込む

ステージごとの回答を受けたら、**次のステージへ進む前に必ず `interview.md` へ追記する**。
書き込む内容は2つ：

1. **Q&A ログ** — ユーザーの回答をそのまま記録する（解釈を混ぜない）
2. **AI の思考メモ** — そのステージで得た情報から、今何を理解し、何が気になっているかを自由に書く。仮説・違和感・気づき・次に確認したいこと、何でもよい

書き込みのタイミング:
- Pre-Stage 回答受け取り → `interview.md` に Pre-Stage を書き込む → Stage 1 へ
- Stage 1 回答受け取り → `interview.md` に Stage 1 を追記 → Stage 2 へ
- Stage 2 回答受け取り → `interview.md` に Stage 2 を追記 → Stage 3 へ
- Stage 3 回答受け取り → `interview.md` に Stage 3 を追記 → Stage 4 へ
- Stage 4 回答受け取り → `interview.md` に Stage 4 を追記 → Stage 5 へ（必要なら）
- Stage 5 回答受け取り（またはスキップ） → `interview.md` に Stage 5 を追記 → Stage 6a へ
- Stage 6a（ディレクター分析）完了・承認 → `interview.md` に Stage 6a を追記 → Stage 6b へ
- Stage 6b（設計方向の対話）完了・承認 → `interview.md` に Stage 6b を追記 → `context.md` / `requirements.md` / `DESIGN.md` を書き出す
