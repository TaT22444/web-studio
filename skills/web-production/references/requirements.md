# 📋 要件定義ガイド（Claude内部参照用）

> このファイルは `/web-studio:setup` の Stage 3〜4 で使用する内部ガイドです。
> Stage 1・2 の内容は `context.md` に書き出し済みのため、ここでは仕様・表現に関する項目のみ扱う。
> 成果物は `{project}/docs/requirements.md` に書き出します。

---

## Stage 3 ヒアリング項目

`[必須]` 項目はすべてユーザーに確認してから記載する。
`[推測可]` 項目はAIが補完し、未確認事項に列挙する。
`[任意]` 項目は決まっていれば聞く。未定の場合はディレクターが提案し、要件定義書に「提案」として記載する。

**ターゲット・CTA**
- `[必須]` 主要ターゲット: 年齢層・職業・ライフスタイルなど
- `[推測可]` ユーザーがサイトに来るときの状況・気持ち
- `[必須]` 取ってほしい行動（CTA）

**デザイン方向性**
- `[必須]` トーン＆マナー: プロフェッショナル / カジュアル / ミニマル など
- `[推測可]` カラー（指定がなければAIが提案し承認を得る）
- `[推測可]` 参考デザインのキーワードやURL

**コンテンツ・構成**
- `[任意]` 必要なページ・セクション（決まっているものがあれば）
- `[任意]` 入れてほしいコンテンツや機能（決まっているものがあれば）
- `[推測可]` 既存素材の有無（テキスト・画像・ロゴなど）

**技術・運用要件**
- `[推測可]` レスポンシブ: PC / SP / 両方
- `[推測可]` 技術スタック（指定がなければAIが提案）
- `[推測可]` 外部連携: フォーム / 決済 / SNS など

---

## Stage 4 で確定する項目

Stage 4（ディレクター分析・構成提案）で、以下が確定またはユーザー承認済みとなる:

- サイトマップ（ページ構成と各ページの役割）
- セクション構成（各ページ内のセクション順序と設計根拠）
- コンテンツ優先度と導線設計

これらを出力フォーマットに反映する。
