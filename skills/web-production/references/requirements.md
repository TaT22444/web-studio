# 📋 要件定義ガイド（Claude内部参照用）

> このファイルはClaudeが Phase 1 を進めるための内部ガイドです。
> ユーザーが編集するものではありません。
> Phase 1の成果物（要件定義書）は `{project-name}/docs/requirements.md` に書き出します。

---

## ヒアリングの進め方

既存資料（議事録・要望書・参考URL等）があればそれを優先して読み込む。
各項目を `[必須]` / `[推測可]` で分類しており、**`[必須]` 項目はすべてユーザーに確認してから要件定義書に記載する。** `[推測可]` 項目は文脈から妥当に推測して記載し、未確認事項として明示してよい。

> ⚠️ **ヒアリング前に記載しないこと。** サイト名・ブランド名を含む `[必須]` 項目は、ユーザーの承認を得る前に要件定義書へ書き込んではならない。

### ヒアリングシート

**1. プロジェクト基本情報**
- `[推測可]` サイト種別: コーポレートサイト / LP / ポートフォリオ / ECサイト / その他
- `[必須]` サイト名・ブランド名（仮案を提示して承認を得ること）
- `[推測可]` 目的・背景: なぜ今このサイトが必要か
- `[推測可]` 新規 / リニューアル

**2. ブランドの本質**（最重要。ここがデザイン・実装全体の軸になる）
- `[必須]` このサイトが存在する理由: 誰のどんな課題を解決するか
- `[必須]` ユーザーに何を感じてほしいか（価値観・感情）
- `[必須]` 絶対にやってはいけないこと（NGトーン・NGカラー・NGイメージ）

**3. ターゲットユーザー**
- `[必須]` 主要ターゲット: 年齢層・職業・ライフスタイルなど
- `[推測可]` ユーザーがサイトに来るときの状況・気持ち
- `[必須]` 取ってほしい行動（CTA）: 問い合わせ / 購入 / 来店 / 資料請求 など

**4. コンテンツ・構成**
- `[必須]` 必要なページ・セクション一覧
- `[推測可]` 既存素材の有無（テキスト・画像・ロゴなど）
- `[推測可]` 参考サイトURL

**5. デザイン方向性**
- `[必須]` トーン＆マナー: プロフェッショナル / カジュアル / ラグジュアリー / ミニマル など
- `[推測可]` カラー: ブランドカラー・NGカラー（指定がなければAIが提案し承認を得る）
- `[推測可]` 参考デザインのキーワードやURL

**6. 技術・運用要件**
- `[推測可]` レスポンシブ: PC / SP / 両方
- `[推測可]` 多言語対応: 不要 / 日英 / その他
- `[推測可]` CMS連携: 不要 / あり（種類）
- `[推測可]` 外部連携: フォーム / 決済 / SNS など

**7. スケジュール**
- `[推測可]` 希望納期（指定がなければ「未定」で記載）
- `[推測可]` フェーズ分け: あり / なし

---

### ヒアリング進行ルール

1. `[必須]` 項目で未確認のものをすべて洗い出し、**まとめて1回の質問**でユーザーに確認する。何度も往復しない。
2. 質問はヒアリングシートの構造をベースにしつつ、自然な会話形式でまとめてよい。ただし項目の**省略・スキップは禁止**。
3. サイト名・ブランド名は仮案を提示し、ユーザーの承認を得てから要件定義書に記載する。
4. ユーザーの回答後、`[推測可]` 項目はAIが補完し、推測した内容は要件定義書の「未確認事項」セクションに列挙する。
